## Лабораторная работа №2

### Система управления автобусными маршрутами с использованием двусвязного списка


#### Обоснование выбора структуры данных

Для реализации автобусного маршрута был выбран **Двусвязный кольцевой список**. Ниже приведено сравнение со стандартным динамическим массивом (`list` в Python):

1. **Естественное моделирование кольцевого маршрута:** В кольцевом списке указатель `next` последнего элемента (хвоста) ссылается на первый элемент (голову), а указатель `prev` головы — на хвост. Это позволяет бесконечно циклически перемещаться по маршруту без применения математических операций остатка от деления (`%`) на каждом шаге.
2. **Эффективный разворот маршрута (Построение обратного пути):** Благодаря наличию указателя `prev` на предыдущую остановку, разворот маршрута выполняется *in-place* (без выделения дополнительной памяти) за линейное время $O(N)$ путём простой перестановки указателей `next` и `prev`. В динамическом массиве разворот требует либо сдвига, либо усложнения логики индексации перегонов.
3. **Эффективное добавление элементов:** Добавление новой остановки в конец маршрута (или в любое другое место) при наличии ссылки на хвост (`tail`) выполняется за константное время $O(1)$, тогда как динамический массив при превышении капасити может инициировать дорогостоящую операцию реалокации памяти и копирования элементов ($O(N)$ в худшем случае).

#### Структура проекта

Код программы спроектирован с разделением на логические слои для обеспечения читаемости, гибкости и тестируемости:

* **Слой моделей:** Класс `BusStop`. Описывает сущность остановки, хранит её состояние (название, координаты, время) и ссылки на соседние узлы.
* **Слой логики:** Класс `BusRoute`. Содержит алгоритмы управления списком остановок, функции расчёта времени, циклической навигации и разворота.
* **Слой ввода-вывода:** Класс `RouteReportGenerator`. Отвечает за представление данных и взаимодействие с файловой системой.
* **Блок тестирования:** Демонстрационный код, проверяющий корректность работы всех функций и обрабатывающий граничные случаи.

#### Документация к функциям и классам

##### 1. Класс `BusStop`
Является узлом двусвязного кольцевого списка.

* `__init__(self, name: str, coordinates: tuple[float, float], time_to_next: float)`  
    **Конструктор класса.** Инициализирует остановку.
    * `name`: Название остановки (строка).
    * `coordinates`: Кортеж из двух чисел с плавающей точкой (широта, долгота).
    * `time_to_next`: Время в минутах до следующей по маршруту остановки.
    * `next` и `prev`: Указатели на следующую и предыдущую остановку соответственно (изначально `None`).

##### 2. Класс `BusRoute`
Управляет структурой двусвязного кольцевого списка.

* `add_stop(self, name: str, coordinates: tuple[float, float], time_to_next: float) -> None`  
    **Добавление остановки.** Создаёт новый узел `BusStop` и вставляет его в конец списка. Автоматически перестраивает кольцевые связи между `tail` и `head`. Если список пуст, зацикливает элемент на самого себя.
    * *Сложность:* $O(1)$
* `calculate_total_time(self) -> float`  
    **Расчёт общего времени.** Проходит по всему кольцу от `head` до `tail` ровно один раз и суммирует время перегонов между всеми остановками. Если маршрут пуст, возвращает `0.0`.
    * *Сложность:* $O(N)$
* `get_location_after_n_stops(self, start_stop_name: str, n: int) -> BusStop`  
    **Поиск остановки через N шагов.** Находит стартовую остановку по имени, а затем перемещается по указателям `next` на $N$ шагов вперёд. 
    * *Граничные случаи:* Если список пуст или $N < 0$ — выбрасывает `ValueError`. Если стартовая остановка не найдена — выбрасывает ошибку.
    * *Оптимизация:* Использует операцию `n % size` для исключения лишних полных кругов при больших значениях $N$.
    * *Сложность:* $O(N)$ (на поиск стартовой позиции) + $O(N \pmod{\text{size}})$ на перемещение.
* `reverse_route(self) -> None`  
    **Построение обратного маршрута.** Разворачивает маршрут *in-place*. Меняет местами указатели `next` и `prev` у каждого узла, корректно пересчитывая времена путей (время пути $A \rightarrow B$ при развороте становится временем пути $B \rightarrow A$). В конце меняет местами ссылки на `head` и `tail`.
    * *Сложность:* $O(N)$

##### 3. Класс `RouteReportGenerator`
Класс для генерации отчётов.

* `save_to_text_file(route: BusRoute, filename: str) -> None`  
    **Формирование подробного отчёта.** Статический метод. Обходит переданный маршрут и записывает структурированную информацию (общее время, координаты, перегоны) в текстовый файл. Корректно обрабатывает пустой маршрут, создавая уведомление в файле.
    * *Сложность:* $O(N)$

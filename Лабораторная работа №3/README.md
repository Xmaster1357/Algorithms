# Задача 1 — Система обработки задач (Queue)

## Описание

Реализована очередь задач.

Каждая задача содержит:

- id
- описание
- приоритет

---

## Класс Task

Используется для хранения информации о задаче.

### Поля:

| Поле        | Описание             |
| ----------- | -------------------- |
| id          | идентификатор задачи |
| description | описание задачи      |
| priority    | приоритет задачи     |

---

## Класс TaskQueue

Реализует очередь задач.

### Методы:

| Метод         | Описание                    |
| ------------- | --------------------------- |
| enqueue(task) | добавление задачи           |
| dequeue()     | удаление первой задачи      |
| front()       | просмотр первой задачи      |
| isEmpty()     | проверка очереди на пустоту |
| show_tasks()  | вывод всех задач            |

---

## Вариативная часть №9

Добавлена проверка на одинаковые id.

Если задача с таким id уже существует — новая задача не добавляется.

# Задача 2 — Проверка выражений (Stack)

## Описание

Реализована проверка корректности скобочной последовательности.

Поддерживаются:

- ()
- []
- {}

---

## Класс Stack

Реализует стек.

### Методы:

| Метод      | Описание                   |
| ---------- | -------------------------- |
| push(item) | добавление элемента        |
| pop()      | удаление элемента          |
| peek()     | просмотр верхнего элемента |
| isEmpty()  | проверка стека             |
| show()     | вывод содержимого стека    |

---

## Функция check_brackets(expression)

Проверяет правильность скобочной последовательности.

### Возможности:

- проверка корректности строки
- обнаружение ошибок
- вывод состояния стека после каждого действия

---

## Вариативная часть №9

После каждого шага отображается содержимое стека.

Пример:

```text
Добавили '(' -> Стек: ['(']
Удалили '(' -> Стек: []
```
